JDBC基础

JDBC-API访问数据库步骤

  • 加载驱动
  • 建立连接，获取Connection
  • 执行SQL语句
  • 关闭连接

加载驱动

  • 加载驱动有两种方法

第一种，最常用的方法使用Class.forName(String deriver)方法加载驱动

第二种，使用JDBC-API管理层的DriverManager类的registerDriver(Driver driver)方法向DriverManager注册给定驱动

建立连接，获取Connection

  • 使用JDBC-API管理层的DriverManager类的getConnection(String url,String name,String passWord);

Connection接口常用方法

方法 功能描述
Statement createStatement() 创建一个Statement对象将SQL语句发送到数据库
PreparedStatement preparedStatement(String sql) 创建一个PreparedStatement对象将参数化的SQL语句发送到数据库
CallableStatement prepareCall(String sql) 创建一个CallableStatement对象调用数据库的存储过程
void commit() 将上一次提交/回滚进行的更改改为持久更改，并释放Connection对象保存的所有数据库锁定
Rollback void rollback() 取消在事务中进行的所有更改，并释放Statement对象保存的所有数据库锁定
Statement createStatement(int resultSetType,int resultSetConcurrency) 创建一个Statement对象，该对象将生成具有给定类型和并发性的ResultSet对象

获取Statement对象的两种方法

注意：

第二种方法中type的值决定结果集的滚动方式，而concurrency的值决定结果集是否可更新

  • type
    • ResultSet.TYPE_FORWARD_ONLY:结果集的指针只能向前移动
    • ResultSet.TYPE_SCROLL_INSENSITIVE:结果集的指针可以滚动通常不受其他更改的影响
    • ResultSet.TYPE_SCROLL_SENSITIVE:结果集的指针可以滚动通常受其他更改的影响
  • concurrency
    • ResultSet.CONCUR_READ_ONLY:结果集不可更新
    • ResultSet.CONCUR_UPDATABLE:可以通过更改结果集中的数据更改数据库中的数据

执行SQL语句

JDBC-API执行SQL提供了三个接口如下：

  • Statement接口——执行静态SQL语句返回结果对象
  • 下面PreparedStatement接口是Statement接口的子接口
    • PreparedStatement接口——预编译SQL语句对象
  • CallableStatement接口继承自PreparedStatement接口
    • CallableStatement接口——执行SQL存储过程的接口
  • 注意，开发时使用PreparedStatement接口，而不使用Statement接口
  • 原因是PreparedStatement的代码可读性和维护性更高，能够最大的提高代码的执行效率，以及能够提供程序的安全性

Statement接口方法

方法 功能描述
ResultSet executeQuery(String sql) 执行给定SQL语句，返回单个ResultSet对象
int executeUpdate(String sql) 执行给定SQL语句，可以是更新、删除、插入
boolean execute(String sql) 执行给定SQL语句，可能返回多个结果
ResultSet getResultSet() 以ResultSet对象形式检索当前结果
int getUpdateCount() 以更新计数的形式检索当前结果
void close() 立即释放Statement对象的数据库和JDBC资源
boolean getMoreResults() 移动到此Statement对象的下一个结果

  • 两种方法：
    • 带参数存储过程

      • 设置参数和PreparedStatement接口设置参数一样
